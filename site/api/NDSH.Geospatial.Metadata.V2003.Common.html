AbstractObject_Type

The AbstractObject_Type class serves as an abstract base type for geospatial metadata objects in compliance with the ISO 19115 standard. It implements the IObjectIdentification interface, which provides unique identification attributes such as id and uuid.

CodeListValueItem

The CodeListValue_Type prototype is used to refer to a specific codelist value in a register.

CodeListValue_Type

Represents a specific value within a registered code list, ensuring standardized categorization and reference. This class implements multiple metadata interfaces to provide comprehensive code list support as per the ISO 19115 metadata standard.

NilReasonModel

Represents a standardized way to document the reason for a missing or null value in geospatial metadata. It is based on the gml:NilReasonType definition in the GML schema and supports enumerated reasons NullEnumeration as well as custom user-defined explanations. The class ensures that nil reasons are properly structured, validated, and serialized for interoperability with XML and JSON-based metadata systems.

ObjectReference_PropertyType

Represents a reference to an external object within the ISO 19115 metadata framework. It provides attributes that define relationships between metadata elements, enabling linkage to remote resources via unique identifiers or hyperlinks. This class incorporates elements from the XLink standard (xlink) and supports referencing via attributes such as href, role, arcrole, and title. Additionally, it implements mechanisms for specifying missing values through the nilReason attribute, following the gml:NilReasonType convention.

Interfaces

ICodeList

Defines a contract for objects that require a code list reference as per the ISO 19115 metadata standard. This interface ensures that implementing classes provide a structured mechanism to reference external code lists.

ICodeListValue

Defines a contract for objects that require a reference to a specific value from an external code list. This interface ensures that implementing classes provide a structured mechanism for referencing standardized code list values, following the ISO 19115 metadata standard.

ICodeSpace

Defines a contract for objects that require a reference to a code space. This interface ensures that implementing classes provide a structured mechanism for referencing a code space as per the ISO 19115 metadata standard.

IId

Defines a contract for objects that require a unique identifier within an XML document. This interface aligns with the ISO 19115 metadata standard and ensures compliance with the gcoBase schema's ID attribute definition.

INilReason

The INilReason interface defines a content model that allows recording of an explanation for a null value or other exception.

IObjectIdentification

Provides a standardized interface for identifying metadata objects with both a unique local identifier and a globally unique UUID, ensuring interoperability across different datasets and systems.

IObjectReference

Defines a contract for referencing objects within geospatial metadata, incorporating linking capabilities and UUID-based references.

IUuid

Defines a contract for objects that require a universally unique identifier (UUID). This interface ensures that objects can be uniquely referenced across different datasets, following the ISO 19115 metadata standard.

IUuidRef

Defines a contract for objects that require a reference to a universally unique identifier (UUID). This interface ensures that implementing classes provide a structured mechanism for referencing an existing UUID in compliance with the ISO 19115 metadata standard.

IValue

Represents an interface for defining a standard value property in geospatial metadata. This interface ensures that implementing classes provide a structured and serializable mechanism for handling values in compliance with the ISO 19115 standard.
