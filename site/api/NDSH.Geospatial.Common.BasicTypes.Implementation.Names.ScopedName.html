Remarks

A ScopedName consists of a namespace (scope) and a local name. It provides a structured naming mechanism that allows metadata elements to be referenced in a hierarchical manner, supporting interoperability in geospatial metadata.

CodeSpace can have values from the following three categories:

  • A simple name (e.g., a dictionary name, authority, or classification scheme).
  • A hierarchical namespace (it can be a dot . pattern, similar to programming namespaces).
  • A URI (Uniform Resource Identifier) (commonly used for globally unique references).

EXAMPLES:

  1. Simple Name
    • "ISO19115"A metadata standard name
    • "EPSG"A spatial reference system authority
    • "OGC"Open Geospatial Consortium
  2. Dot (.) Namespace Pattern
    • "Geospatial.Metadata"A structured namespace in a system
    • "Physical.Geology"A hierarchical category in a geoscience dataset
    • "ISO.Geospatial.Metadata"A structured ISO metadata namespace
  3. URI (Web-based Reference)
    • "https://www.opengis.net/def/namespaces"A persistent web reference to a geospatial definition
    • "http://metadata.iso.org/19115"A direct reference to ISO 19115 metadata
    • "http://www.epsg-registry.org/"The EPSG coordinate reference system registry
  4. URN (Uniform Resource Name)
    • "urn:ogc:def:crs:EPSG::4326"A structured identifier for an EPSG coordinate system
    • "urn:x-ogc:def:metadata:ISO19115:2003"A URN reference to ISO 19115 metadata
  5. UUID-based Identifier
    • "urn:uuid:123e4567-e89b-12d3-a456-426614174000"A globally unique identifier for a metadata record
