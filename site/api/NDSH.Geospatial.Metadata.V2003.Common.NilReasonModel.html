Represents a standardized way to document the reason for a missing or null value in geospatial metadata. It is based on the gml:NilReasonType definition in the GML schema and supports enumerated reasons NullEnumeration as well as custom user-defined explanations. The class ensures that nil reasons are properly structured, validated, and serialized for interoperability with XML and JSON-based metadata systems.

Remarks

Usage:

  • Used in geospatial metadata to indicate why a value is missing or inapplicable.
  • Supports both predefined enumeration values (NullEnumeration) and free-text reasons prefixed with "other:".
  • Can reference an external explanation using a URI.

Key Features:

  • Implements the INilReason interface to standardize the handling of missing values.
  • Ensures valid nil reasons using predefined values or structured custom reasons.
  • Provides static helper methods to retrieve correctly formatted nil reason values.

Properties

NilReason

Gets/Sets an explanation for why a value is missing.

The NilReason is a string value of one of the following three options:

  1. The value is a low case string representation of one of the values of the NullEnumeration, except the value Other
  2. The value is a combination of the low case string representation of the value Other followed by : and then by a string providing a brief explanation, where text is a string of two or more characters with no included spaces.
    Example: other:brief explanation goes here
    The entire string must be valid against the following regular expression:
    other:\w{2,}
  3. The value is a Uniform Resource Identifier (URI).
    A particular community may choose to assign more detailed semantics to the standard values provided by the NullEnumeration. Alternatively, the URI method enables a specific or more complete explanation for the absence of a value to be provided elsewhere and indicated by-reference in an instance document.

Example: 

<pre><code class="lang-csharp">NilReason = INilReason.GetNilReason(NullEnumeration.Missing);
NilReason = INilReason.GetNilReason(NullEnumeration.Other, "Example reason give here");
NilReason = INilReason.GetNilReason("http://www.example.com/a-detailed-explanation-for-nilreason/";</code></pre>

Methods

GetNilReason(NullEnumeration, string)

Generates a NilReason string based on a predefined enumeration value. The method can be used to set the string value of the NilReason property.

The procedure makes sure that rules for setting the string are observed and should be preferably used rather than setting the string value using the property's setter. The reason for having a public setter in the NilReason property is to allow for XML deserialization.

Example: 

<pre><code class="lang-csharp">NilReason = INilReason.GetNilReason(NullEnumeration.Missing);
NilReason = INilReason.GetNilReason(NullEnumeration.Other, "Example reason given here");</code></pre>
