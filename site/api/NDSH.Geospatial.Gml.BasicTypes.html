CodeType

The CodeType is a generalized type to be used for a term, keyword or name. It adds a XML attribute codeSpace to a term, where the value of the codeSpace attribute (if present) shall indicate a dictionary, thesaurus, classification scheme, authority, or pattern for the term.

CodeWithAuthorityType

The CodeWithAuthorityType is a generalized type to be used for a term, keyword or name. It adds a XML attribute codeSpace to a term, where the value of the codeSpace attribute (if present) shall indicate a dictionary, thesaurus, classification scheme, authority, or pattern for the term.

CoordinatesType

The CoordinatesType is deprecated for tuples with ordinate values that are numbers. The CoordinatesType is a text string, intended to be used to record an array of tuples or coordinates. While it is not possible to enforce the internal structure of the string through schema validation, some optional attributes have been provided in previous versions of GML to support a description of the internal structure. These attributes are deprecated. The attributes were intended to be used as follows:

  • Decimal symbolUsed for a decimal point (default="." a stop or period).
  • cs symbol Used to separate components within a tuple or coordinate string (default="," a comma).
  • ts symbol Used to separate tuples or coordinate strings (default=" " a space).
MeasureListType

The MeasureListType provides for a list of quantities, i.e. a collection of measured values and their common unit of measure (UOM).

NilReasonEntity

Represents a standardized way to document the reason for a missing or null value in geospatial metadata. It is based on the gml:NilReasonType definition in the GML schema and supports enumerated reasons NilReasonEnumeration as well as custom user-defined explanations. The class ensures that nil reasons are properly structured, validated, and serialized for interoperability with XML and JSON-based metadata systems.

NilReasonRemoteSchemaEntity

The NilReasonRemoteSchemaEntity class extends the NilReasonEntity and implements the IRemoteSchema interface, providing both a standardized way to document missing or null values and a deprecated reference to an external schema. This class ensures proper handling of nil reasons based on the gml:NilReasonType definition while also maintaining compliance with GML-based schema references.

UomIdentifier

The UomIdentifier defines the syntax and value space of the unit of measure identifier. Represents a Unit of Measure (UOM) Identifier in GML.

UomSymbol

Represents a Unit of Measure Symbol following the "Unified Code of Units of Measure" (UCUM) standard.

UomUri

Represents a URI-based Unit of Measure Identifier.

NilReasonEnumeration

Enumerates standardized reasons for a property or attribute to have a null (missing) value, following the GML (ISO 19136) nil-reason pattern. This provides clarity on why a value is unavailable or not applicable at a given time.

SignType

SignType is a convenience type with values + (plus) and - (minus).
