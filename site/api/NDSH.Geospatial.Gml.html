AbstractGMLType

The AbstractGMLType is any GML object having identity. It includes objects that are GML features, or other objects, with identity. It is effectively an abstract superclass for all GML objects.

AbstractInlineProperty

Represents an abstract in-line property within a GML object.

The AbstractInlineProperty class serves as the head of a substitution group for more specific elements that provide an in-line value.

AbstractMemberType

To create a collection of GML Objects that are not all features, a property type shall be derived by extension from AbstractMemberType. This abstract property type is intended to be used only in object types where software shall be able to identify that an instance of such an object type is to be interpreted as a collection of objects. By default, this abstract property type does not imply any ownership of the objects in the collection. The owns attribute of gml:OwnershipAttributeGroup may be used on a property element instance to assert ownership of an object in the collection. A collection shall not own an object already owned by another object.

AbstractMetaData

Represents the abstract base class for metadata elements in GML.

AbstractMetaDataType

Represents the base class for metadata types in GML.

AbstractMetadataPropertyType

To associate metadata described by any XML Schema with a GML object, a property element shall be defined whose content model is derived by extension from AbstractMetadataPropertyType. The value of such a property shall be metadata. The content model of such a property type, i.e. the metadata application schema shall be specified by the GML Application Schema. By default, this abstract property type does not imply any ownership of the metadata. The Owns property (owns attribute) of the IOwnership (gml:OwnershipAttributeGroup) may be used on a metadata property element instance to assert ownership of the metadata.

If metadata following the conceptual model of ISO 19115 is to be encoded in a GML document, the corresponding Implementation Specification specified in ISO/TS 19139 shall be used to encode the metadata information.

AssociationAndOwnership

Represents a generalized association in GML, providing an XLink-based mechanism for referencing external or internal resources within geospatial metadata. This abstract class implements the GML AssociationAttributeGroup pattern from gmlBase.xsd, enabling a standard approach for linking GML objects and facilitating interoperability with ISO 19115:2003 metadata structures.

Encoding a GML property in-line vs by-reference shall not imply anything about the “ownership” of the contained or referenced GML Object, i.e. the encoding style shall not imply any “deep-copy” or “deep-delete” semantics.To express ownership over the contained or referenced GML Object, the gml:OwnershipAttributeGroup attribute group may be added to object-valued property elements.If the attribute group is not part of the content model of such a property element, then the value may not be “owned”. When the value of the owns attribute is true, the existence of in-line or referenced object(s) depends upon the existence of the parent object.

GenericMetaDataType

Represents a generic metadata element in GML.

The GenericMetaDataType class allows extensible metadata content in GML. It supports mixed content, meaning it can hold both text and structured XML elements.

GmlObject

The GmlObject serves as a common root for all complex content elements in GML.
Represents the abstract base class for all objects that participate in the GML substitution group.

InlinePropertyType

Represents a property that provides an in-line value for a GML object.

The InlinePropertyType class is used to represent GML objects that must be provided in-line rather than by reference. It contains an instance of AbstractObject, which defines the actual object.

MetaDataPropertyType

Represents a metadata property in GML, which allows associating metadata with a GML object. The metadata can be in-line or a reference to external metadata.

ReferenceType

ReferenceType is intended to be used in application schemas directly, if a property element shall use a 'by-reference only' encoding.

StringOrRefType

Represents a string value or a reference to an external resource in GML.

This type allows a property to:

  • Store a simple text string.
  • Contain an external reference to another resource using XLink attributes.
It is commonly used for descriptions, identifiers, and other metadata elements.

IAssociation

Defines a contract for geospatial metadata associations, incorporating linking capabilities, nil reasons, and optional references to remote schemas. This interface aligns with the AssociationAttributeGroup definition from the GML schema.

IId

Defines a contract for GML objects that require a unique identifier. This interface aligns with the GML standard and ensures compliance with the gmlBase schema's ID attribute definition.
