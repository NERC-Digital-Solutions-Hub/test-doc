Namespace NDSH.Geospatial.Gml.Coordinate.Systems

Classes

AbstractCoordinateSystemType

The AbstractCoordinateSystemType is a coordinate system that is the non-repeating sequence of coordinate system axes that spans a given coordinate space. A coordinate system is derived from a set of mathematical rules for specifying how coordinates in a given space are to be assigned to points. The coordinate values in a coordinate tuple shall be recorded in the order in which the coordinate system axes associations are recorded. This abstract complex type shall not be used, extended, or restricted, in an Application Schema, to define a concrete subtype with a meaning equivalent to a concrete subtype specified in this document.

AffineCSType

The AffineCSType is a two- or three-dimensional coordinate system with straight axes that are not necessarily orthogonal. An AffineCSType shall have two or three gml:axis property elements; the number of property elements shall equal the dimension of the coordinate system.

CartesianCSType

The CartesianCSType is a 1-, 2-, or 3-dimensional coordinate system.

  • In the 1-dimensional case, it contains a single straight coordinate axis.
  • In the 2- and 3-dimensional cases gives the position of points relative to orthogonal straight axes. In the multi-dimensional case, all axes shall have the same length unit of measure.
A CartesianCSType shall have one, two, or three gml:axis property elements.
CoordinateSystemAxisType

The CoordinateSystemAxisType is a definition of a coordinate system axis.

CylindricalCSType

The CylindricalCSType is a three-dimensional coordinate system consisting of a polar coordinate system extended by a straight coordinate axis perpendicular to the plane spanned by the polar coordinate system. A CylindricalCSType shall have three gml:axis property elements.

EllipsoidalCSType

The EllipsoidalCSType is a two- or three-dimensional coordinate system in which position is specified by geodetic latitude, geodetic longitude, and (in the three-dimensional case) ellipsoidal height. An EllipsoidalCSType shall have two or three gml:axis property elements; the number of associations shall equal the dimension of the CS.

LinearCSType

The LinearCSType is a one-dimensional coordinate system that consists of the points that lie on the single axis described. The associated coordinate is the distance – with or without offset – from the specified datum to the point along the axis. A LinearCSType shall have one gml:axis property element.

ObliqueCartesianCSType

The ObliqueCartesianCSType is a coordinate system that is based on two straight axes that are not necessarily orthogonal.

PolarCSType

The PolarCSType is a two-dimensional coordinate system in which position is specified by the distance from the origin and the angle between the line from the origin to a point and a reference direction. A PolarCSType shall have two gml:axis property elements.

SphericalCSType

The SphericalCSType is a three-dimensional coordinate system with one distance measured from the origin and two angular coordinates. A SphericalCSType shall have three gml:axis property elements.

TemporalCSType

The TemporalCSType is a temporal coordinate system in which time is used as a single axis for positioning.

TimeCSType

The TimeCSType is a one-dimensional coordinate system containing a time axis, used to describe the temporal position of a point in the specified time units from a specified time origin. A TimeCSType shall have one gml:axis property element.

UserDefinedCSType

The UserDefinedCSType is a two- or three-dimensional coordinate system that consists of any combination of coordinate axes not covered by any other coordinate system type. A UserDefinedCSType shall have two or three gml:axis property elements; the number of property elements shall equal the dimension of the coordinate system.

VerticalCSType

The VerticalCSType is a one-dimensional coordinate system used to record the heights or depths of points. Such a coordinate system is usually dependent on the Earth's gravity field, perhaps loosely as when atmospheric pressure is the basis for the vertical coordinate system axis. A VerticalCSType shall have one gml:axis property element.

Interfaces

IGeocentricCoordinateSystem

The IGeocentricCoordinateSystem is used to define coordinate systems that are used in three-dimensional spatial reference frames and are used in earth-centred coordinate systems.

IGeodeticCoordinateSystem

The IGeodeticCoordinateSystem is used to mark classes that define geodetic coordinate reference systems.

IImageCoordinateSystem

The IImageCoordinateSystem is used to define planar or grid-based coordinate systems commonly used in engineering or image-related coordinate reference systems.

ITemporalCoordinateSystem

The ITemporalCoordinateSystem represents a temporal coordinate system used within a temporal coordinate reference system.
